It shall also be the main means by which anything can change the state of the entity. Besides a few exceptions, Components should only hold data and methods that alter and/or accesses those data. \doxysubsection{Detailed Description}
class interface for all Components owned by any Entity.\+Components shall store attributes and data related to their respective owner \mbox{\hyperlink{classba_1_1Entity}{Entity}}. It shall also be the main means by which anything can change the state of the entity. Besides a few exceptions, Components should only hold data and methods that alter and/or accesses those data. The library should provide many different components for most common usage, but if a user needed to create a custom cumponent that could interact with the systems of this library, the user need to define a public static constant of unsigned integer type named CID to ID the component for efficiency reasons.

Called once after the entity owner is added to the main loop. 

This \mbox{\hyperlink{classba_1_1Component_a6f0b77ccea25915379aa4b81bd29e25c}{awake()}} does nothing, but subtypes should override this to ensure data is instantiated properly and the owner entity has the prerequisites needed to use that component. i.\+e. \mbox{\hyperlink{classba_1_1Animation}{Animation}} component needed \mbox{\hyperlink{classba_1_1Sprite}{Sprite}} component to work.
